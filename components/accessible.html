Component Redesign: Search Bar

Overview

It is crucial for web components to incorporate accessibility into user interactions for different forms of input (mouse, keyboard) and output (visual cues, labels for screen readers). I looked at the accessibility of search bars and redesigned one to make it more accessible.

Application Research

I started by researching Safari, Spotify, and Apple Mail's search bar components.

SafariInputsOutputs
Mouse & Touchpad can click (double, triple), drag, highlight, but search icon does not click click highlights outline and cursor, screenreader gives instructions and reads out letters
Keyboard copy paste highlight shortcuts, esc to exit, ⌘+F to enter focus order slow to navigate to search results
Touch tap (double and triple), x to navigate out tap gives you options to share, select
SpotifyInputsOutputs
Mouse & Touchpad can click (double, triple), drag, highlight, search icon clicks click/hover highlights outline, screenreader does not read what you type or highlight
Keyboard copy paste highlight shortcuts, esc deletes, ⇧+⌘+K to search focus order good to go to search results but slow to navigate back
Touch tap (double and triple), cancel button or swipe up to navigate out tap gives you option to autofill
MailInputsOutputs
Mouse & Touchpad can click (double, triple), drag, highlight, but search icon does not click click highlights outline and cursor, screenreader says “toolbar,” not search, reads out as you type
Keyboard copy paste highlight shortcuts, esc to exit, tab to enter focus order goes to mailboxes, not to results
Touch tap (double and triple), x or swipe up to navigate out tap and release tap lightens search bar

I found that the search bars I researched all had strengths and weaknesses. I combined the best features into a redesigned Safari search bar.

State Model

The first state model demonstrates Safari's current search box.

The second state model adds a hover effect for mouse users, increasing efficiency. It also adds a keyboard shortcut for keyboard users, increasing accessibility.

Final Redesign

I redesigned Safari's search bar based off my state models.

Adding a keyboard shortcut increases accessibility, but I chose to decrease learnability as it is not a commonly used shortcut.

Similarly, while adding a hover effect is more efficient for mouse users, this is not a feature keyboard users benefit from.

Next Steps

The changes I added are only a small part of increasing accessibility. The search bars I observed all have good support for keyboard actions, mouse clicks, and highlighting. Additional changes include increasing screen- reader support and changing font size according to system settings.
